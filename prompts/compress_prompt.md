# Context 压缩摘要模板

## 用途

当对话消息数超过阈值（{threshold} 条）时，将早期消息压缩为摘要，
作为 system 消息注入到消息列表最前面，确保 LLM 能"回忆"之前的话题。

## 摘要格式

[对话历史摘要] 以下是之前对话的关键信息总结：

### 用户话题
{user_topics}

### 工具使用记录
{tool_uses}

### 关键信息
{key_decisions}

## 压缩规则

1. 用户话题：提取每条用户消息的前 {topic_max_chars} 个字符
2. 工具使用：记录工具名称和关键参数
3. 关键信息：提取 assistant 回复中的数字、结论、重要判断
4. 压缩后保留最近 {keep_recent} 条消息以保证连贯性
5. 不调用 LLM，直接规则提取，零延迟
